## Описание системы Альфа-банка

### Альфа-Онлайн
| Поле | Значение |
|--------|--------|
| Название системы | Альфа-Онлайн |
| Категория | System of Engagement (SoE) |
| Роль в бизнесе | Основной цифровой канал для клиентов банка. Предоставляет доступ к банковским продуктам через iOS, Android и веб-браузер. Отображает данные из операционных систем, принимает команды от клиента. |
| Входящие потоки данных | REST API (HTTPS/JSON) от Клиента; REST API (HTTPS/JSON) от API Gateway (ответы на запросы) |
| Исходящие потоки данных | REST API (HTTPS/JSON) в API Gateway (запросы операций); REST API (HTTPS/JSON) в MDM (клиентские данные); REST API (HTTPS/JSON) в Loyalty (запросы баллов и кэшбэка) |

### CRM (отделения)
| Поле | Значение |
|--------|--------|
| Название системы | CRM (отделения) |
| Категория | System of Engagement (SoE) |
| Роль в бизнесе | Рабочее место сотрудника отделения и колл-центра. Построена на Java / Kotlin / Spring. Позволяет сотруднику видеть профиль клиента, открывать продукты, вести заявки и историю обращений. |
| Входящие потоки данных | REST API (HTTPS/JSON) от Сотрудника; REST API (HTTPS/JSON) от API Gateway; REST API (HTTPS/JSON) от MDM (профиль клиента); HTTPS/SOAP/XML от Core Banking |
| Исходящие потоки данных | REST API (HTTPS/JSON) в API Gateway; REST API (HTTPS/JSON) в Сотрудника (ответы); HTTPS/SOAP/XML в Core Banking (команды открытия продуктов); REST API (HTTPS/JSON) в DWH (события взаимодействий) |

### API Gateway
| Поле | Значение |
|--------|--------|
| Название системы | API Gateway |
| Категория | Внешний |
| Роль в бизнесе | Единая точка входа для всех внешних запросов. Выполняет авторизацию, маршрутизацию запросов к внутренним сервисам, rate limiting и логирование. Не содержит бизнес-логики. |
| Входящие потоки данных | REST API (HTTPS/JSON) от Альфа-Онлайн; REST API (HTTPS/JSON) от CRM (отделения) |
| Исходящие потоки данных | REST API (HTTPS/JSON) в MDM; REST API (HTTPS/JSON) в Core Banking; REST API (HTTPS/JSON) в Loan Management; REST API (HTTPS/JSON) в Loyalty |

### Core Banking
| Поле | Значение |
|--------|--------|
| Название системы | Core Banking |
| Категория | System of Record (SoR) |
| Роль в бизнесе | Главный источник правды банка. Хранит все счета, транзакции и финансовые проводки. Является ядром IT-ландшафта — все системы либо читают из него, либо инициируют в нём операции. Работает 24/7. |
| Входящие потоки данных | HTTPS/SOAP/XML от CRM (команды открытия счетов); HTTPS/SOAP/XML от API Gateway; REST API (HTTPS/JSON) от Card Processing (результаты транзакций) |
| Исходящие потоки данных | HTTPS/SOAP/XML в CRM (данные счетов); SQL/ETL в DWH; Kafka/MQ в Антифрод (поток транзакций)  |

### MDM (клиенты)
| Поле | Значение |
|--------|--------|
| Название системы | MDM (клиенты) |
| Категория | System of Record (SoR) |
| Роль в бизнесе | Централизованный реестр мастер-данных о клиентах. Единственный источник правды об идентификационных данных: ФИО, паспорт, контакты, KYC-статус. Все системы, которым нужны данные о клиенте, обращаются сюда. |
| Входящие потоки данных | REST API (HTTPS/JSON) от API Gateway; HTTPS/SOAP/XML от ЕСИА / Госуслуги (подтверждение личности); REST API (HTTPS/JSON) от CRM |
| Исходящие потоки данных | REST API (HTTPS/JSON) в Альфа-Онлайн; REST API (HTTPS/JSON) в CRM; SQL/ETL в DWH |

### DWH
| Поле | Значение |
|--------|--------|
| Название системы | DWH |
| Категория | System of Insight (SoI) |
| Роль в бизнесе | Централизованное хранилище исторических данных банка (Greenplum / Tableau). Консолидирует данные из всех операционных систем для аналитики и отчётности. Не влияет на операционные процессы напрямую. |
| Входящие потоки данных | SQL/ETL от Core Banking; SQL/ETL от MDM; Kafka/MQ от Антифрод |
| Исходящие потоки данных | SQL Query в Analytics (сегменты, скоринг); SQL Query в BI-дашборды Tableau |

### Антифрод
| Поле | Значение |
|--------|--------|
| Название системы | Антифрод |
| Категория | System of Insight (SoI)  |
| Роль в бизнесе | Система противодействия мошенничеству на основе ML-моделей (real-time). Анализирует каждую транзакцию в режиме реального времени, выявляет аномалии. Может инициировать блокировку транзакции до вывода средств. |
| Входящие потоки данных | Kafka/MQ от Core Banking (поток транзакций); Kafka/MQ от Card Processing (карточные операции) |
| Исходящие потоки данных | REST API (HTTPS/JSON) в Card Processing (блокировка); Kafka/MQ в DWH (данные инцидентов) |
